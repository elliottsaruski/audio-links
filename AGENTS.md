## What This Project Is

An audio-first link-in-bio platform for musicians, producers, DJs, and audio creatives. Think Linktree's simplicity combined with SoundCloud's audio playback. Each creator gets a single shareable public page at `/{handle}`.

**MVP question:** Can a musician create an account, upload tracks, customize a profile, and share one clean page? If yes, MVP succeeds.

**GitHub:** https://github.com/elliottsaruski/audio-links

## Supabase Infrastructure

### Database Tables

**`profiles`**
- `id` (uuid, FK to auth.users)
- `handle` (text, UNIQUE)
- `display_name`, `bio` (max 150 chars), `avatar_url`, `background_url`
- `location`, `accent_color`, `card_color`, `text_color`, `wrapper_color`

**`tracks`**
- `id`, `user_id` (FK), `title`, `artist`
- `audio_url`, `cover_url`, `peaks` (JSONB)
- `has_audio` (bool), `description` (text), `track_links` (JSONB)
- `is_pinned` (bool), `sort_order` (int)

**`links`**
- `id`, `user_id` (FK), `platform` (predefined list), `url`, `sort_order`

**`upcoming_releases`**
- `id`, `user_id` (FK), `title`, `release_date`, `presave_url`, `cover_url`

### Storage Buckets

| Bucket | Allowed types | Size limit |
|---|---|---|
| `audio` | audio/* | 50MB |
| `covers` | image/* | 5MB |
| `avatars` | image/* | 5MB |
| `backgrounds` | image/* | 10MB |

**Storage path convention:** `{bucket}/{user_id}/filename` вЂ” always include the user_id segment.

### RLS
All tables have Row Level Security enabled. Users can only read/write their own rows. Public profile data (`profiles`, `tracks`, `links`, `upcoming_releases`) is readable by anyone.

## Architecture Decisions

### Dashboard
- Desktop-only (no mobile layout needed)
- Single-page tabbed editor: Profile, Tracks, Releases, Links
- Live phone frame mockup preview reads from Zustand store (not live DB)
- No forced onboarding wizard вЂ” all tabs accessible immediately after handle selection

### Public Profile Page (`/[handle]`)
- Track cards show: cover art, title, artist only вЂ” **no waveform on cards**
- Sticky player bar fixed at bottom вЂ” always visible once a track is loaded
- Waveform renders **only** in the sticky player bar via Wavesurfer.js
- Pinned track loads into the player by default on page load

### Audio Pipeline
1. User selects file in dashboard
2. Mediabunny extracts waveform peaks client-side ("Analyzing audio..." state shown)
3. Audio file + peaks + metadata uploaded to Supabase
4. Public profile loads peaks from DB, Wavesurfer renders from peaks (no re-processing)

### Constraints
- 10 tracks per user (hard cap вЂ” enforce in UI and DB)
- 50MB audio file limit
- 150-character bio limit
- Social link platforms are a predefined list (not freeform): the platform field is an enum
- Per-track streaming links (stored as JSONB in `track_links`) support: Spotify, Apple Music, Tidal, YouTube, SoundCloud, Bandcamp

---

## Auth Flow

1. User signs up в†’ Supabase sends confirmation email
2. User clicks link в†’ `/auth/callback` route handler
3. Redirects to `/dashboard`
4. Middleware at root `middleware.ts` protects `/dashboard` вЂ” unauthenticated users redirected to `/auth/login`
